# 登录页重设计 Spec：精密门厅方向

## 目标

将 `web-console` 的登录页重设计为一张具有展示属性的技术平台门厅页面，使用户在进入系统前即可明确感知：

- 这是一个 `多维度质量评价 AI 平台`
- 平台底座由 `Go 微服务` 与 `Python 模型能力` 协同支撑
- 登录动作是进入统一控制台的正式入口，而不是普通后台表单

本次设计以视觉与信息架构升级为主，不改变当前真实登录逻辑、认证状态管理和登录后跳转行为。

## 设计结论

采用 `B. 精密门厅` 方向，并按以下关键词统一设计：

- 精密
- 前沿
- 克制
- 展示型
- 技术平台感先行

该方向不使用监控大屏式高压迫感，也不走纯营销官网风，而是在“高级企业级 AI 平台入口”与“电力工业技术底座表达”之间取得平衡。

## 页面定位

登录页的首要任务不是展示运行状态，而是建立平台认知。

页面需要先让用户看到 `Go + Python + AI 平台基础设施`，再理解这套平台的核心价值是 `多维度质量评价`。因此信息层级遵循：

1. 技术平台感先行
2. 质量评价能力随后展开
3. 登录动作保持明确、专注、低干扰

## 信息架构

页面采用 `左 60 / 右 40` 的双栏结构，在窄屏下退化为上下单栏布局。

### 左侧：技术平台展厅

左侧负责建立品牌、平台属性和技术认知，采用从上到下的三层结构。

#### 1. 顶部品牌区

- 英文小标：用于增强平台化、产品化气质
- 中文主标题：围绕 `多维度质量评价 AI 平台`
- 一段短说明：明确该平台由 `Go 微服务编排 + Python 模型推理与评分能力` 共同驱动

建议的表达目标：

- 看起来像一套高级技术平台的正式入口
- 不像监控页，不像普通管理后台，也不像轻量营销页

#### 2. 中部展示模块区

中部使用 3 个展示模块，不做平均分布的“功能卡堆砌”，而做成更有层次的展厅式模块。

模块一：`Go 微服务编排`

- 传达认证、任务、资产、审计、网关等服务协同关系
- 重点表达平台是“有架构、有边界”的系统
- 展示语义偏“服务编排、链路协作、统一入口”

模块二：`Python 模型推理与评分`

- 传达生成、推理、评价、分析由模型链路承接
- 重点表达平台不是静态展示系统，而是有真实 AI 能力的执行底座
- 展示语义偏“模型运行、结果分析、推理链路”

模块三：`多维度质量评价能力`

- 传达平台支持多指标、多场景、多结果维度的质量评价
- 重点表达该平台的业务核心，而不是泛化的 AI 门户
- 展示语义偏“评价框架、质量判断、结果解释、任务可追溯”

#### 3. 底部关键词摘要带

左侧底部增加一条轻量级技术摘要带，用于收尾和强化品牌识别。

建议关键词方向：

- Service Orchestration
- Model Inference
- Quality Evaluation
- Audit Trace

这一层用于补强“展示感”，不承载复杂说明。

### 右侧：登录门厅

右侧负责承接登录动作，视觉上是一个更挺拔、更克制的精密面板。

建议结构如下：

- 标题：`进入评价控制台` 或 `登录 AI 评价平台`
- 简短说明：说明登录后进入统一工作台
- 错误提示区：保留清晰、即时的失败反馈
- 表单区：账号、密码、登录按钮
- 辅助说明区：默认演示账号与密码保留，但降级为次要信息
- 底部轻提示：说明登录后进入生成、评分、审计等统一工作流

右侧的设计原则是：

- 动作明确
- 信息简洁
- 与左侧展厅形成“展示 / 进入”分工

## 需要弱化或移除的现有内容

以下现有内容不适合保留在新版主视觉中：

- 偏运行监控语义的 `Gateway / Model Runtime / Asset Pipeline / Online` 状态板
- `G:\\Runtime` 这类过于底层、过于本地化的运行时路径表达
- “本机原生优先”这类偏环境说明、但不够产品化的标签文案

它们的问题不是技术上错误，而是会破坏新版首页的“高端产品门厅”气质，使页面重新滑回“运维监控页”。

如需保留相似含义，应改为更产品化、更抽象的表达，例如：

- 统一控制台
- 可信接入
- 服务协同
- 模型驱动

## 视觉语言

### 整体气质

视觉方向定义为：

- 高级企业级 AI 平台
- 精密、冷静、秩序感强
- 少装饰，多结构
- 有技术辉光，但不赛博朋克

页面不应依赖厚重玻璃拟态，也不应使用明显的大屏仪表板样式。核心是通过背景层次、边框、排版、光效和色彩节制来体现高级感。

### 背景

- 使用统一的深冷基底，而非明显割裂的双色分栏渐变
- 可叠加极轻微的精密网格、细线框或低对比结构纹理
- 背景存在微弱辉光，但辉光是辅助，不是主角

### 卡片与面板

- 登录卡片采用 `低反射、薄边框、轻悬浮` 的精密面板风格
- 左侧展示模块边界清晰，但避免普通后台式厚重卡片
- 通过透明度、描边、明暗层级建立秩序，而不是通过大量阴影制造“悬浮感”

### 排版

- 标题需要有强层级和足够留白
- 说明文字克制、简短、可信
- 左侧信息更像展厅陈列，不像说明书
- 右侧表单更像正式入口，不像演示页

## 色彩系统

整体色彩采用 `冷深海军蓝 + 电光青 + 雾银灰 + 少量琥珀`。

### 色彩目标

- 用深色建立技术平台的稳重与前沿感
- 用电光青承担聚焦、交互和 AI 技术感
- 用雾银灰维持页面秩序和精密感
- 用极少量琥珀为“评价、标注、判断”提供高级点缀

### 推荐色板

- `--bg-900: #081421`
- `--bg-800: #0D1B2A`
- `--panel-700: #13263A`
- `--line-500: rgba(120, 154, 188, 0.24)`
- `--text-100: #EAF4FF`
- `--text-300: #9FB0C3`
- `--accent-500: #35D6FF`
- `--accent-600: #27C7E8`
- `--warm-500: #F4B860`

### 色彩分配比例

- 70% 深色背景与深色面
- 20% 灰蓝中性色
- 8% 电光青强调
- 2% 琥珀点缀

### 页面应用建议

- 页面背景：`--bg-900` 到 `--bg-800` 的深冷层次
- 标题文字：接近冷白的 `--text-100`
- 正文与辅助信息：`--text-300`
- 分隔线、细框、模块轮廓：`--line-500`
- 主按钮、输入焦点、重点标签：`--accent-500` / `--accent-600`
- 少量关键词标记、状态点缀：`--warm-500`

## 文案方向

### 主叙事

主叙事从“工业控制中心”调整为“多维度质量评价 AI 平台”，但仍保留技术底座的清晰表达。

建议文案目标：

- 先让用户看到平台是完整的技术体系
- 再让用户理解平台的核心价值在于质量评价与分析

### 文案语气

- 专业
- 克制
- 正式
- 不口号化
- 不运维告警化

### 表达原则

- 避免“Ready / Online / Active”堆砌
- 避免本地路径、环境偏好、部署细节成为视觉中心
- 强调平台能力、协同链路和评价价值

## 交互与行为约束

本次设计只改界面表现，不改现有真实登录行为。

必须保留：

- `http.post('/auth/login', form)`
- `authStore.setSession(...)`
- 成功提示与失败提示
- 登录中 loading 状态
- 按 `redirect` 或默认 `/dashboard` 跳转

错误提示需要同时具备：

- 卡片内可见的错误反馈
- 即时消息反馈

## 响应式要求

- 桌面端：双栏结构，左侧展示、右侧门厅
- 平板与窄屏：自动切换为上下结构
- 登录区在窄屏下仍保持完整留白与可读性
- 左侧展示内容在窄屏下可以压缩层级，但不能挤成普通卡片列表

## 成功标准

设计完成后，登录页需要达到以下结果：

- 第一眼能看出这是技术平台入口，而不是普通后台登录页
- 能清楚感知 `Go + Python + AI` 的平台底座
- 能明确理解平台核心是 `多维度质量评价`
- 登录区域聚焦、专业、可信
- 页面整体更像“高端产品门厅”，而不是“运行状态看板”

## 实施边界

本 spec 不要求：

- 改动后端服务
- 增加新的前端 UI 框架
- 改动认证流程
- 扩展额外业务功能

本 spec 仅定义登录页的展示重设计方向，为后续实现计划提供基准。
