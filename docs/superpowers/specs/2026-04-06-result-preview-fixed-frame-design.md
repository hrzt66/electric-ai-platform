# Result Preview Fixed Frame Design

**日期**：2026-04-06

## 目标

调整生成工作台中间区域的结果展示方式，使生成后的图片只显示在“多图预览”卡片内部，不再视觉上铺满整个中间列；同时保留点击查看大图的能力。

## 设计决策

采用“固定预览框 + 点击弹层大图”方案。

- 中间列整体布局保持不变。
- 仅调整 `ResultPreview.vue` 的主图展示方式。
- 主图被限制在卡片内部的固定展示框中，使用 `contain` 保持完整比例。
- 缩略图切换区域保留在主图下方。
- 点击主图时，仍允许使用 Element Plus 的图片预览弹层查看大图。

## 涉及文件

- 修改：`web-console/src/components/workbench/ResultPreview.vue`
- 可能微调：`web-console/src/views/GenerateView.vue`

## 具体行为

### 空状态

- 当还没有生成结果时，空状态仍只显示在“多图预览”卡片内部。

### 有结果时

- 主图展示在卡片内部的固定高度预览框中。
- 图片按比例缩放，不裁切主体，不向外扩张布局。
- 下方继续展示任务号、图片名、总分等元信息。
- 多图任务继续通过底部缩略图切换当前主图。

### 点击主图

- 点击主图后，允许调用 Element Plus 的预览能力查看大图。
- 大图预览只作为显式点击后的辅助查看，不再作为默认铺满式展示。

## 样式约束

- 不改变生成工作台三列结构。
- 不改变右侧评分雷达与实时状态布局。
- 不增加新的全屏覆盖层逻辑，继续复用 `el-image` 自带预览能力。

## 风险与规避

- 风险：固定高度过小会导致纵向图过于局促。
  规避：使用相对稳妥的固定高度区间，并保持 `contain`。

- 风险：预览框过高仍会造成“像铺满中间列”的视觉效果。
  规避：收紧主图高度，让图片区、元信息区、缩略图区的层次更明显。

## 验证方式

- 前端单测：`npm --prefix web-console run test`
- 前端构建：`npm --prefix web-console run build`
- 页面人工检查：提交生成任务后，确认图片仅出现在“多图预览”卡片内；点击主图后可以正常弹出大图预览。
