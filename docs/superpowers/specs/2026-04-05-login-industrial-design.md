# 工业控制中心风登录页设计

## 目标

将当前 `web-console` 登录页升级为符合工业电力 AI 平台定位的入口页面，使它既能承担真实登录职责，也能作为平台门面展示微服务架构、模型中心能力和运行时特征。

## 设计方向

采用 `A 方案：工业控制中心风`。

- 左侧为平台叙事区，强调工业电力、模型中心、运行时和微服务协同。
- 右侧为登录舱，强调进入工作台的唯一入口与当前运行形态。
- 整体视觉沿用现有蓝灰 + 铜金体系，但提升层次、对比与仪表感。

## 信息架构

### 左侧叙事区

- 平台主标题：工业电力图像生成与评分平台
- 副说明：突出变电站、巡检机器人、设备数字孪生、工业场景合成
- 三块能力卡片：
  - Go 微服务
  - Python 模型中心
  - 统一运行时
- 背景需具备工业控制中心气质，而不是普通后台渐变。

### 右侧登录舱

- 入口说明：登录平台
- 运行标签：本机原生优先
- 错误反馈：登录失败时给出清晰提示
- 表单字段：账号、密码
- 主按钮：进入工作台

## 视觉语言

- 深蓝钢质背景搭配铜金强调色
- 使用大面积非纯色背景，避免普通白卡片堆叠感
- 登录卡片采用半透明或轻玻璃效果，但仍需保证文本可读性
- 字体层级要明显，页面应一眼看出“这是平台入口”
- 移动端应自动退化为上下结构，避免横向拥挤

## 交互要求

- 保留当前真实登录逻辑，不改接口行为
- 保留登录中 loading 状态
- 登录成功后仍按 `redirect` 或默认 `/dashboard` 跳转
- 登录失败时展示卡片内错误提示并弹消息

## 约束

- 不破坏已有认证状态管理
- 不引入新的 UI 框架
- 保持与现有工作台壳层风格一致，但入口页要更强视觉记忆点

## 验证方式

- `npm --prefix web-console run build`
- 登录页源文件不应再出现乱码文案
- 登录页在桌面与窄屏布局下都应保持完整结构
